# @beyou/contracts

TypeScript types generated from the BeYou backend's SpringDoc OpenAPI 3.1.0 spec.

## What this package exposes

- `paths` — all API route types (request/response shapes per method)
- `components` — all shared schema types
- `operations` — all named operations
- `Schemas` — shorthand alias for `components['schemas']`

The canonical types are in `src/schema.d.ts`, generated by `openapi-typescript` from the committed `openapi.json` snapshot.

## Refreshing from the live backend

The live endpoint is **auth-protected**, so generation runs from a committed snapshot — not a live unauthenticated fetch.

To update `openapi.json` from a running backend:

```bash
# 1. Start the backend (dev profile)
cd Beyou-backend-spring && ./mvnw spring-boot:run

# 2. Obtain a bearer token (log in as a verified user)
TOKEN=$(curl -s -X POST http://localhost:8099/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"test@test.com","password":"123456"}' \
  | grep -o '"accessToken":"[^"]*"' | cut -d'"' -f4)

# 3. Fetch and overwrite the snapshot
curl -s -H "Authorization: Bearer $TOKEN" \
  http://localhost:8099/api/v1/v3/api-docs \
  -o packages/contracts/openapi.json

# 4. Regenerate TypeScript types
npm --workspace @beyou/contracts run generate
```

## Scripts

| Script | Purpose |
|--------|---------|
| `generate` | Regenerate `src/schema.d.ts` from `openapi.json` |
| `check` | Drift gate — fails if `schema.d.ts` is stale vs `openapi.json` |
| `typecheck` | Compile-time validation of all types including `assert.ts` guard |

The `check` script is the drift gate: it regenerates types to a temp file and diffs against the committed `schema.d.ts`. Run it in CI to catch spec drift before it reaches production.
